# Hello, I am Lorem

`Hello, I am Lorem` был простым учебным приложением с приветствием по имени. Теперь это атмосферный статический showcase-проект: он анализирует имя, подбирает настроение взаимодействия, хранит локальную историю сессий и готов к деплою на GitHub Pages.

## Что стало лучше

- Полностью новый интерфейс с выразительной визуальной подачей и адаптивной вёрсткой.
- Несколько режимов взаимодействия: `Фокус`, `Креатив`, `Спокойствие`, `Смелость`.
- Генерация цифрового профиля имени: приветствие, архетип, аура, рекомендации и зеркальная версия имени.
- Локальная история сессий через `localStorage` и мини-аналитика на первом экране.
- Отдельная страница `Quality Lab` для просмотра JSON-отчёта тестов.
- PWA-база: `manifest.webmanifest` и `service-worker.js` для офлайн-кэширования статических файлов.
- Подготовленный GitHub Actions workflow для публикации на GitHub Pages.

## Стек

- HTML5
- CSS3
- Vanilla JavaScript
- Jest + jsdom
- GitHub Pages

## Структура

- `src/` — всё, что реально публикуется как статический сайт
- `src/index.html` — главная студия приветствий
- `src/results.html` — панель качества
- `src/scripts/` — браузерная логика и рендеринг страниц
- `src/styles/` — дизайн-система и стили страниц
- `src/assets/` — иконки и статические ассеты
- `src/service-worker.js` — офлайн-кэш приложения
- `src/manifest.webmanifest` — PWA-манифест
- `tests/` — unit/integration тесты
- `docs/screenshots/` — скриншоты проекта
- `.github/workflows/deploy.yml` — автодеплой на GitHub Pages

## Локальный запуск

1. Установить зависимости:

```bash
npm install
```

2. Запустить локальный статический сервер:

```bash
npx serve src
```

3. Открыть приложение в браузере:

```text
http://localhost:3000
```

## Тесты

Обычный прогон:

```bash
npm test
```

Прогон с генерацией JSON-отчёта для `src/results.html`:

```bash
npm run test:report
```

## Деплой на GitHub Pages

1. Создай репозиторий на GitHub и запушь проект в ветку `main`.
2. В настройках репозитория открой `Settings -> Pages`.
3. Убедись, что source настроен на `GitHub Actions`.
4. После пуша workflow из `.github/workflows/deploy.yml` сам опубликует сайт.
